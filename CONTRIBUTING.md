# Contributing to LLM Engineer Toolkit

First off — thanks for taking the time to contribute! 🎉

## What to Add

✅ **Good contributions:**
- New prompt templates (with real use cases)
- New patterns (RAG variants, agent types, chains)
- New code examples that are production-quality
- Resources (papers, tools, courses) — must be free or have free tier
- Bug fixes in existing code

❌ **Don't add:**
- Broken or untested code
- Paid-only resources without mention
- Duplicate content

## How to Contribute

1. **Fork** the repo
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test your code** — make sure it runs!
5. **Submit a PR** with a clear description

## Code Style

- Python: follow PEP8, use type hints
- Add docstrings to all functions
- Keep examples self-contained and runnable
- Comment complex logic

## PR Template

```
## What does this add?
[Brief description]

## Type
- [ ] New pattern
- [ ] New prompt template
- [ ] New resource
- [ ] Bug fix

## Tested?
- [ ] Yes, it runs
```
